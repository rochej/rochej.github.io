Map Yap - 12/13/2015

While many of Ruby’s built-in enumerable methods are specifically designed to sort the items in the collection by different parameters, map is a powerful tool to act on each item in a collection.

Just the other day, I was sitting with a group of dancers writing some thank-you notes for donations we’d received. We were passing the notes around the table, taking turns adding different things to each note until each one was done. Please suspend your disbelief and imagine that our Ruby code would work in the physical world.


notes=[Phonecia’s note, David’s note, Jak’s note, Britknee’s note, Ed’s note, …]
notes.map!{ | note | add salutation} #Roche’s code
notes.map!{ | note | add info about last project and next project} #Virginia’s code
notes.map!{ | note | add happy holidays!} #Sam’s code
notes.map!{ | note | address the envelope} #Alex’s code
notes.map!{ | note | stick on a stamp} #Odessa’s code

Can you imagine how long it would take to write code for each of my dancer friend’s to do each task? And what if we forgot to do one task on one note? If we forgot the stamp, the note would never get to its recipient!

Also, notice above that we are using the destructive .map! rather than .map. The later would create a new array, while the former acts directly on the original array.

Now let’s use this method on some actual code.


array = [1,2,3,4,5,6,7,8]
array.map {|item| print “#{item}! “ if item%2==0}
print “Who do we appreciate?!?”

Would display: 2! 4! 6! 8! Who do we appreciate?!?

Our .map iterates over the array and prints out only the numbers that are even—very useful for our spirit day cheer.
