Javablog - 01/01/2016

Happy new year!

I am happy to report that both Ruby and Javascript have metaobjects. This means that rather than defining basic properties and methods for each and every object created over and over again, it is possible to make a template that can be applied when needed.

Imagine that we wanted to make some beverages for our New Years Eve extravaganza. Making a template object would be a good start.

In Ruby, these metaobjects are called classes—described last week.

class Beverage

attr _accessor :booze, :mixer, :strength

def initialize (booze, mixer, strength)
  @booze = booze
  @mixer = mixer
  @strength = strength
end

def how_crazy?
  if @booze ==“tequila” && @strength >5
    puts “Hella crazy!”
  else
    puts “OK.”
  end
end

end

In JavaScript, these metaobjects are established through constructor functions.

var Beverage = function (booze, mixer, strength) {
  this.booze=booze;
  this.mixer=mixer;
  this.strength=strength;
  this.how_crazy = function (){
    if (booze == "tequila" && strength > 5)
      console.log("Hella crazy!");
    else
      console.log("OK.");
  }
};

var margarita= new Beverage ("tequila", "lime", 6)

margarita.how_crazy()
=>Hella crazy!

There are a few dissimilarities between these two strategies. Notice the syntactical differences--Ruby's instance variables are demarcated by an @, wheras Javascript uses this. More significantly, Ruby’s classes come with many build in methods and can have a parent class as well, while one must write ANY methods needed for a JavaScript constructed object.
